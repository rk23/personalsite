Whetstone

Whetstone was built with a Node/Express server, MongoDB persistence, the Materialize CSS library, and jQuery for animations and Ajax routing. The point is to keep you accountable for some sort of creative/personal goal you set. It's like doing those 30/60 day challenges but with more motivators to get to it. We have the option to invite friends and set a daily "incentive" price. If you don't submit proof of whatever the goal was, then you pay that incentive to the pot and it gets shared among all your friends at the end of your goal.


We deliberately chose not to make it a SPA with Angular because there weren’t many actions required by the user – simple login/logout, goal creation, submission entry and viewing submissions was about all that’s required of the user so adding Angular would have weighed down the app a lot without providing that much value to the end user.


The biggest design challenge we faced was the synchronisity of making a submission by a certain time and the asychrony of Node. We solved this by using an asynchronous control flow package along with a chron job to keep track of the time.


At the point of writing, the splash page copy hasn't been written yet. Most functionality will have to be seen by registering. It is currently still in development but you can find it here for now:

whetstone-dev.us-west-2.elasticbeanstalk.com ↗

Parse

Parse was built in under a week and a half by a team of four of us with Ruby on Rails and jQuery with Foundation as the CSS library. It takes a block of text, applies bucket sort to it to count all the unique words, returns the most frequent words in a descending list as well as highlights the words according to their frequency.


I was responsible for implementing the backend routing, Ajax routing and state change, implementing the algorithm, and frequency highlighting.


I’ve been trying to learn languages for years and the idea came from a frustration with most teaching sources giving verb conjugation lists for you to memorize. Most of those conjugations you never see in real contexts. By using this system, you get a vocab list of the most used words in any language in real contexts, like a newspaper or short story. It’s intended for people who have an intermediate understanding of a language but need to find that vocabulary to study.


You can find it at: parse-lexicon.herokuapp.com ↗
